# Quarterly Access Review Checklist (Template)

## Objective
Validate that user access remains appropriate for role, employment status, and least-privilege requirements.

## Systems in Scope
- [ ] System/App 1
- [ ] System/App 2
- [ ] Privileged access systems (if applicable)

## Review Steps
### 1) Prepare
- [ ] Export current user access list per system
- [ ] Confirm list includes: user, role, entitlements, last login, manager, department
- [ ] Identify privileged/admin accounts separately

### 2) Manager/Owner Review
- [ ] Send access list to designated reviewer(s)
- [ ] Reviewer confirms: keep / remove / modify
- [ ] Document exceptions with justification + expiration date

### 3) Remediation
- [ ] Remove access no longer needed
- [ ] Modify roles/permissions where required
- [ ] Validate completion (screenshots/logs/ticket closures)

### 4) Evidence & Audit Readiness
- [ ] Retain exports, reviewer responses, tickets, and approvals
- [ ] Record review completion date and attestation
- [ ] Store evidence in approved repository/location

## Evidence Checklist
- [ ] Exported access list(s)
- [ ] Reviewer approval/attestation
- [ ] Exception log (with expiry)
- [ ] Tickets showing removal/modification
- [ ] Completion summary
