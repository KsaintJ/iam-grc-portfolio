# Access Review Workflow (Quarterly)

## Overview
This diagram illustrates the periodic access review process used to validate that user access remains appropriate, authorized, and aligned with least-privilege principles.

The workflow supports audit readiness by ensuring reviewer accountability, documented decisions, remediation tracking, and evidence retention.

---

## Access Review Flow

Access Inventory Generated  
→ Reviewer Assigned (Manager / System Owner)  
→ Access Evaluated (Keep / Modify / Remove)  
→ Exceptions Documented (with justification & expiration)  
→ Remediation Executed (Access Removed or Updated)  
→ Verification Completed  
→ Evidence Retained for Audit

---

## Key Control Objectives
- Ensure access aligns with current job role
- Detect and remove excessive or stale access
- Enforce reviewer accountability
- Maintain defensible audit evidence

---

## Evidence Artifacts
- Access export reports
- Reviewer attestations
- Exception logs
- Tickets showing access removal/modification
- Completion summary
